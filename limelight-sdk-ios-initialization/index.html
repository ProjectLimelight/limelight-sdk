    Limelight iOS SDK¶

    Limelight is the main entry point of the Limelight iOS SDK. It provides SDK initialization, runtime configuration, and debug controls. All setup should happen at app launch, before any ads are requested.

    Initialization¶

    Call Limelight.initialize(host:completion:) as early as possible — typically in application(_:didFinishLaunchingWithOptions:).

    // AppDelegate.swift
import LimelightSDK

func application(_ application: UIApplication,
                 didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {

    Limelight.initialize(host: "https://ads.example.com") { result in
        switch result {
        case .success:
            print("Limelight SDK initialized")
        case .failure(let error):
            print("Initialization failed: \(error)")
        }
    }

    return true
}

    The completion callback is optional. If you do not need to react to the initialization result, you can omit it:

    Limelight.initialize(host: "https://ads.example.com")

    Runtime Configuration¶

    Configure these static properties before or after initialization. Changes take effect on the next ad request.

    Property Type Default Description
    adRequestTimeout Int (ms) 2000 HTTP timeout for ad requests
    loadAdTimeout TimeInterval (s) 6.0 Time allowed for a creative to load
    loadPrecachedAdTimeout TimeInterval (s) 30.0 Time allowed for video/interstitial creatives to load 
    Limelight.adRequestTimeout = 3000
Limelight.loadAdTimeout = 8.0
Limelight.loadPrecachedAdTimeout = 45.0

    Custom Targeting¶

    Attach custom targeting data to all ad requests via the targeting property.

    let targeting = LimelightTargeting()
targeting.userAge = 25
targeting.keywords = ["sports", "tech"]

Limelight.targeting = targeting

    Debug Configuration¶

    Log Level¶

    Control the verbosity of SDK logs:

    Limelight.logLevel = .verbose  // development
Limelight.logLevel = .error    // production

    Test Mode¶

    Enable test mode to mark all bid requests as test traffic (sets bidRequest.test = 1):

    Limelight.testMode = true  // never enable in production builds

    Privacy & Consent¶

    Consent signals (TCF, CCPA, GPP) are read automatically from UserDefaults written by your CMP. The only signal configured via code is COPPA, set on Targeting.shared:

    Targeting.shared.subjectToCOPPA = true

    See the Consent Management guide for full details.

    SDK Version Info¶

    print(Limelight.sdkVersion)    // e.g. "1.0.0"
print(Limelight.omsdkVersion)  // e.g. "1.4.2"

    Complete Setup Example¶

    func application(_ application: UIApplication,
                 didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {

    // Debug (remove in production)
    Limelight.logLevel = .verbose
    Limelight.testMode = true

    // Runtime tuning
    Limelight.adRequestTimeout = 3000
    Limelight.loadPrecachedAdTimeout = 45.0

    // Targeting
    let targeting = LimelightTargeting()
    targeting.keywords = ["news", "finance"]
    Limelight.targeting = targeting

    // Initialize
    Limelight.initialize(host: "https://ads.example.com") { result in
        if case .failure(let error) = result {
            print("SDK init error: \(error)")
        }
    }

    return true
}
